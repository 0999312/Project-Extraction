# Minimal Vector 主题设计

> 版本 0.1 – 2026-03-30

## 1. 概述

`minimal_vector.tres` 是项目当前的**默认全局 UI 主题**。它在 `project.godot` 的 `[gui] theme/custom` 中被指定，因此标准 Godot 控件会默认继承该主题，除非场景或脚本在本地进行了样式覆写。

这个主题的目标是提供一种**高对比、易读、具有风格化矢量感的 UI 外观**，核心特征包括：

- 强烈的黑色描边
- 使用纯色平面，而不是贴图皮肤
- 明亮、偏街机风格的强调色
- 在不同菜单中保持一致的按钮交互语言

## 2. 作用范围

该主题当前覆盖以下通用控件族：

| 控件 | 主题覆盖内容 |
|---|---|
| `Button` | 完整状态样式（`normal`、`hover`、`pressed`、`focus`、`disabled`） |
| `Label` | 字体颜色 + 描边 |
| `LineEdit` | normal/focus/read-only 表面、光标、选区 |
| `Panel` | 通用暖色面板表面 |
| `PanelContainer` | 通用容器表面 |
| `ProgressBar` | 背景 + 填充 |
| `TabContainer` | 面板 + 标签页状态 |
| `HScrollBar` / `VScrollBar` | 轨道 + 拖块状态 |

## 3. 本主题**不负责**的部分

一些游戏内专用 UI 元素刻意使用**脚本级 `StyleBoxFlat` 覆写**，而不是完全依赖全局主题：

- 物品栏网格单元（`scripts/game/ui/inventory_slot.gd`）
- HUD 快捷栏格子（`scripts/game/ui/player_hud.gd`）
- 物品栏菜单中的快捷栏格子与装备占位槽（`scripts/game/ui/inventory_menu.gd`）

这样划分是有意为之：

- `minimal_vector.tres` 负责**共享的应用级皮肤**
- 游戏交互槽位视觉保持**局部、显式、便于按功能微调**

如果要修改背包格子或快捷栏外观，应优先检查这些脚本，而不是直接改全局主题。

## 4. 核心设计目标

### 4.1 优先保证可读性

大部分主要控件采用以下组合：

- 粗黑描边
- 明显分离的前景/背景
- 在高饱和背景上依然可读的文字描边

### 4.2 扁平矢量外观

这个主题避免使用贴图式装饰，整体视觉由以下元素构成：

- `StyleBoxFlat`
- 粗轮廓线
- 纯色填充
- 仅在需要柔和感的地方使用圆角

### 4.3 稳定的按钮交互语言

按钮被视为一套**锁定的视觉交互语言**：

- **normal**：白底 + 黑边
- **hover / focus**：亮绿色底
- **pressed / disabled**：红色系底

最近的配色整理明确保持了按钮颜色不变，只调整了非按钮表面颜色。

## 5. 配色结构

当前配色不是单一色阶，而是按 UI 角色进行组织。

| UI 角色 | 代表值 | 设计意图 |

| 按钮按下/禁用 | `Color(0.9, 0, 0, 1)` | 强动作感 / 阻止状态 |
| Panel 表面 | `Color(0.8588, 0.4706, 0.0706, 1)` | 暖橙棕结构背景 |

| 选中标签页 | `Color(1, 0.8392, 0.2392, 1)` | 高饱和黄色激活强调 |

## 6. 形状语言

### 6.1 边框

大多数主要交互面和容器面都使用**6 px 边框**，这形成了整个 UI 粗描边、卡通/矢量化的轮廓风格。

滚动条轨道相对更轻，使用 **1 px 边框**。

### 6.2 圆角

该主题对圆角采取有选择的使用：

- 按钮：`8 px`
- 输入框：`8 px`
- 进度条：`8 px`
- 标签页面板：上方 `6 px`，下方 `8 px`

这样既保留了轻松感，又不会让所有表面都显得一样柔软。

### 6.3 文字描边

文字可读性高度依赖描边：

| 控件 | 描边尺寸 |
|---|---|
| `Button` | `10` |
| `Label` | `6` |
| `LineEdit` | `10` |
| `ProgressBar` | `10` |

## 7. 分控件说明

### 7.1 Button

按钮是整个主题里最“定型”的部分，修改时应格外谨慎。

- 白色 = 空闲
- 绿色 = 悬停 / 聚焦
- 红色 = 按下 / 禁用
- 黑色描边始终保持不变

如果后续再进行主题重整，只有在**明确重做整套交互语言**时才应调整按钮配色。

### 7.2 Panel 与 PanelContainer

`Panel` 与 `PanelContainer` 故意采用**不同表面颜色**：

- `Panel` 更偏暖橙棕，承担结构背景角色
- `PanelContainer` 更偏绿色，承担内容容器角色

这样即使不使用渐变和贴图，也能让嵌套层级更容易区分。

### 7.3 TabContainer

标签页是非按钮控件里色彩编排最强的一组：

- 蓝/青色内容区域
- 蓝色静止标签
- 琥珀色悬停
- 黄色选中标签

这使当前激活页签非常容易被扫视识别。

### 7.4 ProgressBar

默认进度条被设计为优先保证可读性，之后再按具体玩法做局部改色。例如 HUD 内的状态条可以继续复用相同的轮廓/边框语言，再在脚本中改写填充色。

### 7.5 LineEdit

输入框刻意保持比面板更浅的底色，以保证文本输入时始终清晰可读。青色选区提供了足够反馈，同时不会完全遮挡文字。

## 8. 与游戏内 UI 的关系

游戏内 UI 继承了相同的大方向审美，但**并不是完全由主题资源驱动**：

- 快捷栏格子使用本地黑边 `StyleBoxFlat`
- 物品栏单元格使用本地方形 `StyleBoxFlat`
- 装备占位槽使用本地深色底 + 状态边框

也就是说，项目当前实际上有两层 UI 样式系统：

1. **全局通用控件主题** —— `minimal_vector.tres`
2. **游戏交互样式** —— 脚本动态生成的样式盒

未来可以考虑把部分游戏 UI 样式进一步收敛到主题中，但这不是当前设计的一部分。

## 9. 修改准则

修改 `minimal_vector.tres` 时建议遵循以下规则：

1. **除非明确要重做交互语言，否则不要修改按钮配色。**
2. 若要贴近新的参考风格，优先调整**非按钮表面**。
3. 保留黑色描边与高对比度，这是主题识别度的核心。
4. 主题结构或配色变更后，同时更新英文和中文文档。
5. 如果视觉问题只发生在背包/快捷栏/装备槽，先检查对应游戏 UI 脚本，而不是直接改全局主题。
