# 第三阶段落地任务清单
> 阶段目标：完成“安全区闭环 + 背包升级 + NPC/任务/交易占位”  
> 当前日期：2026-03-12

---

# 1. 阶段目标定义

第三阶段的目标是把项目从“仅有最小战局闭环”推进到“战局内外双循环雏形”。

第二阶段结束时，项目已经具备：
- 进入战局
- 搜刮基础容器
- 与基础敌人交战
- 通过撤离点完成结算

第三阶段结束后，项目应新增以下能力：

1. 存在一个可进入的**安全区占位场景**
2. 玩家在战局结束后可返回安全区
3. 安全区内至少存在：
   - 仓库存取占位
   - 1 个 NPC
   - 1 个任务入口
   - 1 个商人入口
4. 背包系统从“简化搜刮容器”升级为**可扩展的正式背包基础版**
5. 存在最小任务链：
   - 接任务
   - 在战局中完成条件
   - 回安全区交付
6. 存在最小交易流程：
   - 打开交易界面
   - 买入/卖出测试物品
7. 战局结果能影响战局外状态：
   - 获得物品
   - 更新任务进度
   - 更新仓库

# 2. 本阶段范围

## 2.1 必做内容
- 安全区场景与状态流转
- 基础仓库/玩家背包结构
- 最小正式 ItemInstance 模型
- 最小任务系统
- 最小 NPC 对话入口
- 最小交易系统
- 战局结果写回安全区数据
- 主流程从“菜单 -> 战局”升级为“菜单 -> 安全区 -> 战局 -> 结算 -> 安全区”

## 2.2 可简化内容
- 背包先做基础正式版，不要求完整终版 grid inventory 的全部高级规则
- 任务只做单条链路
- 交易只做固定价格与固定库存
- 对话只做基础分支或单段文本
- 安全区只做单房间或单小地图
- 仓库 UI 可先做简化版

## 2.3 不包含内容
- 完整科技树
- 农业
- 工业自动化
- 商人库存刷新机制
- 阵营系统
- 高级对话分支系统
- 多 NPC 复杂关系
- 高级装备改装系统
- 完整状态效果体系
- 多地图与动态事件

---

# 3. 阶段成果定义

第三阶段结束时，项目应具备一个最小双循环：

1. 玩家进入安全区
2. 在安全区接一个任务
3. 整理基础物品或查看仓库
4. 从安全区进入战局
5. 在战局中搜刮/击杀/撤离完成任务条件
6. 结算后返回安全区
7. 与 NPC 对话交任务
8. 可打开商人界面进行基础买卖

# 6. 验收标准

## 6.1 功能验收
必须满足：
- 安全区可进入
- 可在安全区与 NPC 对话
- 可接至少一个任务
- 可打开商人界面
- 可查看玩家背包与仓库
- 战局结果可写回仓库和任务进度
- 可回安全区交任务
- 可再次进入战局

## 6.2 架构验收
必须满足：
- 安全区逻辑由 `SafeZoneController` 组织
- 任务逻辑由 `QuestService` 或同类 Runtime 服务管理
- 交易逻辑由 `TradeService` 管理
- 物品运行时统一使用 `ItemInstance`
- 没有为赶进度把安全区功能直接硬塞进全局单例

## 6.3 体验验收
至少满足：
- 玩家能理解“安全区准备 -> 进入战局 -> 带回资源 -> 回安全区推进任务/交易”的基本循环
- UI 信息足以支持演示和内部测试
- 任务/交易/仓库流程不混乱

# 8. 关键实现建议

## 8.1 背包系统先升级“数据模型”，再升级“表现形式”
第三阶段最重要的是：
- 统一 `ItemInstance`
- 统一玩家背包与仓库数据结构
- 打通战局结果写回

不要一开始就沉迷完整 grid inventory 终版 UI。

## 8.2 任务系统先做“事件驱动最小链路”
优先监听：
- `enemy_killed`
- `item_extracted`
- `raid_finished`

而不是先做复杂任务条件编辑器。

## 8.3 交易系统先做固定库存与固定价格
保证流程通畅，后续再叠加：
- 声望
- 折扣
- 刷新
- 动态经济

## 8.4 安全区先是功能中心，不是内容大地图
第三阶段安全区不需要大而复杂，只需要：
- 承载系统入口
- 串联双循环
- 提供调试与验证平台

# 9. 第三阶段完成后的下一步

建议第四阶段优先进入以下方向之一：

### 方向 A：背包终版化
- 完整 grid-based inventory
- 物品尺寸/旋转/容器嵌套
- 装备栏/快捷栏深化

### 方向 B：系统深化
- 科技树
- 更复杂任务链
- 多 NPC
- 更正式安全区 UI

### 方向 C：战局深化
- 更多敌人类型
- 更复杂地图
- 更正式掉落与随机生成
- 更稳定的战斗反馈

如果以产品演示为目标，建议先走 **方向 A + 部分方向 B**。
