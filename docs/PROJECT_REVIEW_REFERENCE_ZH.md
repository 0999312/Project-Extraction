# 项目人工审查参考

> 用于人工审查整个项目的最简参考文档。

## 1. 审查顺序

1. 项目配置与插件集成
2. 注册表数据与内容定义
3. 核心玩法状态与运行时逻辑
4. 实体、场景与流程
5. UI、输入与本地化
6. 现有文档一致性

## 2. 审查内容范围

| 步骤 | 区域 | 审查内容 | 关键路径 |
|---|---|---|---|
| 1 | 配置层 | 确认启动场景、自动加载、启用插件、输入动作、全局主题是否正确。 | `project.godot`, `override.cfg`, `addons/` |
| 2 | 注册表数据 | 检查物品、武器、抛射物、Buff、实体、标签定义是否完整，ID 是否与运行时使用保持一致。 | `resources/registries/`, `scripts/game/registry/`, `addons/mc_game_framework/` |
| 3 | 玩法系统 | 审查战斗、移动、背包、装备、AI、玩家状态逻辑是否正确，是否存在耦合过深或缺失边界情况。 | `scripts/game/components/`, `scripts/game/systems/`, `scripts/game/projectiles/` |
| 4 | 运行时流程 | 核对场景加载、玩家/敌人初始化、关卡/状态切换、运行时编排是否一致。 | `scripts/game/gameplay/`, `scripts/game/entities/`, `scenes/game_scene/`, `prefabs/entity/` |
| 5 | UI / 本地化 | 审查 HUD、物品栏、暂停/菜单流程、输入映射，以及中英文文本一致性。 | `scripts/game/ui/`, `scenes/`, `resources/i18n/` |
| 6 | 文档 | 检查设计文档与进度文档是否和当前实现一致，双语文档是否成对维护。 | `docs/` |

## 3. 最简输出要求

每一步仅记录：

- **状态：** 通过 / 有问题 / 需跟进
- **主要发现：** 最多 3 条
- **处理动作：** 立即修复 / 延后 / 需澄清需求

## 4. 提示词模板

### 4.1 新增功能 —— 设计提示词

在实现前先使用：

```text
基于 `docs/PROJECT_REVIEW_REFERENCE_ZH.md`，先为以下功能输出设计方案，暂时不要直接实现：

功能：
<描述功能>

需求：
<列出需求>

请仅输出：
1. 审查范围
2. 受影响模块/文件
3. 设计方案
4. 实现顺序
5. 风险点 / 待确认问题
```

### 4.2 新增功能 —— 实现提示词

在设计确认后使用：

```text
请在当前仓库中实现已经确认的功能设计。

功能：
<描述功能>

已确认的设计约束：
<粘贴已确认的设计结论>

需求：
<列出需求>

请：
1. 按 `docs/PROJECT_REVIEW_REFERENCE_ZH.md` 中的审查顺序处理
2. 做最小必要修改
3. 如有需要同步更新相关文档
4. 输出本次修改的文件和实现内容
```

### 4.3 功能不符合要求 —— 反馈与修改提示词

在功能验收不通过时使用：

```text
请对照预期需求检查当前已实现功能，并修复不符合要求的部分。

功能：
<描述功能>

预期需求：
<列出需求>

当前问题 / 审查反馈：
<列出评审意见或不符合预期的点>

请：
1. 指出哪些需求尚未满足
2. 列出受影响的文件/模块
3. 完成必要修改
4. 总结修复内容和剩余待跟进事项
```
