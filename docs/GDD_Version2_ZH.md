# Project Extraction (开发代号) — 游戏设计文档 (GDD) v0.3 (中文)

**日期:** 2026-03-16  
**主要模式:** 单人游戏 (离线，本地权威模拟)  
**未来模式:** 客户端-服务器 **合作 PvE** (内容与规则同单人游戏)  
**核心支柱:** 搜刮与背包决策 → 风险管理 → 撤离 → 安全屋进度 (家园系统) → 科技解锁 (情报)

---

## 1) 核心概念

一款2D俯视角撤离射击游戏，具备类似《逃离塔科夫》的搜刮机制和强化的“搜索并保全”循环。每次Raid发生在一个大型开放地图上，散布着兴趣点(POI)。玩家探索、战斗、搜刮并撤离。在Raid之外，玩家利用从Raid中回收的材料和“情报”来发展一个**安全屋家园系统** (包括建造、生存改善、农业、轻工业/自动化、贸易设施及地图扩展)。

**角色风格:** 人类角色高度抽象且可爱：实际上是一个“会说话的球体加上双手”。这降低了动画复杂度，提高了清晰度和性能，同时保留了战术玩法的深度。

---

## 2) 内容ID：全面采用ResourceLocation (MSF要求)

设计和实现中的所有ID均使用 **Minecraft-Style-Framework `ResourceLocation`** (`命名空间:路径`) 作为唯一标识来源，应用于：
- 注册表条目 (物品、兴趣点、实体、场景、战利品表、任务等)
- 标签
- 事件总线事件
- 存档数据引用
- ECS生成指令

**示例ID:**
- `core:registry/item` (注册表类型RL引用)
- `game:item_weapon` (主要物品类别定义)
- `game:poi_metro_station`
- `game:loot_table_city_crates`
- `game:quest_find_lab_notes`
- `game:tag/weapon_smg` (标签ID作为RL)
- `game:intel_lab_notes`

> **重要:** 根据您的规则，武器/医疗用品等物品注册为主要类别。子类型通过**标签**和数据字段表达。

## 3) 玩法循环

### 3.1 Raid循环 (对局)
1. 从仓库选择装备 (风险 vs 回报)。
2. 部署到Raid地图 (生成的开放区域 + 兴趣点)。
3. 探索兴趣点，搜刮容器，与敌人战斗。
4. 收集材料 + 情报物品 + 任务目标。
5. 移动到撤离点；在撤离倒计时/条件下存活。
6. 撤离 → 将保全的物品转移到仓库；应用任务进度和元奖励。
7. 失败 → 失去未保全的物品 (安全箱除外)。

### 3.2 安全屋家园系统循环 (元游戏)
- 管理仓库 (网格背包)。
- 建造/升级家园设施：存储、制作、医疗站、农场地块、工坊、电网 (轻度)。
- 农业生产 → 加工。
- 轻工业自动化 → 稳定生产关键消耗品/组件。
- 贸易设施 (固定站点，非NPC实体)。
- 通过升级/解锁扩展可进入的安全屋/地图区域。
- 利用情报 (Raid专属知识物品) 解锁科技树。

## 4) 关键系统 (类《逃离塔科夫》功能清单)

### 4.1 玩家状态 (轻量化伤害)
聚焦于可读且与玩法相关的状态：
- `HP` (生命值，单一池)
- `Stamina` (耐力)
- `Encumbrance` (负重，影响速度、耐力回复、噪音)
- `BleedLight` / `BleedHeavy` (轻度/重度流血)
- `Pain` (疼痛，影响瞄准晃动/互动速度)
- 可选后期加入：`Fracture` (骨折，移动惩罚)

**治疗物品:** 注册为**一个主要类别** `game:item_med`。  
效果由标签驱动：`game:tag/med_bandage`、`game:tag/med_painkiller`等。

### 4.2 战斗 (远程 + 近战)
**远程武器**
- 口径 + 开火模式由标签决定 (例如 `game:tag/caliber_9x19`、`game:tag/firemode_auto`)
- 基础后坐力/散布模型
- 基础护甲 vs 穿透模型 (轻量化)

**近战**
- 轻/重攻击
- 短距离弧形/矩形命中
- 耐力消耗

### 4.3 搜刮与容器 (核心支柱)
- 世界容器：板条箱/抽屉/保险箱/尸体/储物柜
- **延迟战利品生成**：首次打开容器时 (或玩家进入区块时) 生成容器内容，以提升性能和可重玩性。
- 声音/风险：打开容器可能发出噪音事件 (可选)。

### 4.4 网格背包
- 玩家背包区域：口袋/战术胸挂/背包/安全箱
- 物品字段：`size_w` (宽)、`size_h` (高)、`weight` (重量)、`max_stack` (最大堆叠)
- 旋转物品
- 堆叠/拆分
- 装备槽位 + 武器模组槽位 (标签 + 槽位规则)

### 4.5 敌人 (人类 + 非人类)
- 人类敌人：抽象的“球体人”，使用远程武器，具备简单战术。
- 非人类敌人：独特的移动/攻击模式 (蜂群、冲锋者、无人机、感染体)。
- 精英/Boss 可选，待核心循环稳定后加入。

### 4.6 任务、对话、交易
- 任务：击杀/获取/放置/侦察/携带物品撤离
- 对话：分支选项，任务门槛
- **无商人实体**：交易者是安全屋中的**固定交互点** (以及可选在Raid内的售货亭)。
- 交易：购买/出售/以物易物，受声望/科技门槛限制

## 5) 数据与内容实现 (具体方案)

本节描述“物品、实体、兴趣点等”如何以与MSF注册表 + GECS ECS兼容的方式存在于游戏中。

### 5.1 注册表 (ResourceLocation驱动)
推荐的注册表类型 (通过MSF中的`RegistryManager.get_registry(type_name)`获取):
- `core:item` → **物品定义** (仅主要类别)
- `core:tag` → 指向某个注册表类型的标签 (MSF `TagRegistry` 需要注册表类型RL)
- `core:poi` → 兴趣点定义
- `core:loot_table` → 战利品表定义
- `core:quest` → 任务定义
- `core:dialogue` → 对话定义
- `core:scene` → 场景模板定义 (安全屋/Raid模板)
- `core:entity_archetype` → ECS原型生成定义 (映射层)

> 注意：MSF `Tag` 存储 `registry_type: ResourceLocation` 和条目作为RL字符串。标签必须指向正确的注册表类型RL (例如 `core:item`)。

### 5.2 物品数据模型 (主要类别 + 标签)
**注册表条目:** `ItemInfo` (MSF演示中`ItemInfo`持有`scene`或`script`以及显示名称)。  
本项目可扩展或包装为一个更丰富的数据资源 (概念示例):
- `id: ResourceLocation`
- `display_name_key: String` (用于国际化)
- `icon: Texture2D`
- `world_drop_scene: PackedScene` (可选：物品掉落世界时使用)
- `size_w: int`, `size_h: int`
- `weight: float`
- `max_stack: int`
- `use_action: ResourceLocation` (例如 `game:use/med_apply`)
- `data: Dictionary` 用于类别特定字段 (弹药属性、耐久度等)
- tags：存储在TagRegistry中 (不放在物品条目内)

**主要类别ID (示例):**
- `game:item_weapon` (所有枪械 + 近战武器共享此主要注册表条目类型；子类型通过标签区分)
- `game:item_med`
- `game:item_ammo`
- `game:item_armor`
- `game:item_material`
- `game:item_tool`
- `game:item_food`
- `game:item_intel`
- `game:item_quest`

**子类型标签 (示例):**
- `game:tag/weapon_smg`、`game:tag/weapon_rifle`、`game:tag/weapon_melee`
- `game:tag/caliber_9x19`、`game:tag/caliber_12g`
- `game:tag/med_bandage`、`game:tag/med_hemostat`、`game:tag/med_painkiller`
- `game:tag/intel_tier1`、`game:tag/intel_lab`

### 5.3 背包实现 (物品在运行时的存在形式)
背包中的物品应为**纯数据**，而非节点，以提升性能和简化存档。

**建议的运行时结构:**
- `ItemStack`:
  - `item_id: ResourceLocation`
  - `count: int`
  - `durability: float` (可选)
  - `custom_data: Dictionary` (剩余弹药、附件列表RL等)
  - `tags_cache: Array[ResourceLocation]` (可选优化)
- `GridInventory`:
  - `width`, `height`
  - 占用格子映射 → 堆叠ID/索引
  - 每个堆叠的放置信息 (`x`, `y`, `rotated`)

**世界掉落物:** 当物品被丢弃时，生成一个**世界拾取实体** (见6.2)，引用`item_id`。

### 5.4 兴趣点数据模型 (开放世界 + 片段)
兴趣点注册表条目包含：
- `poi_id: ResourceLocation`
- `scene_fragment: PackedScene` (视觉效果 + 碰撞 + 标记点)
- `footprint: Rect2i` 或 多边形定义 (用于放置检查)
- `biome_tags: Array[ResourceLocation]`
- `loot_profile_id: ResourceLocation` (战利品表RL)
- `spawn_profile_id: ResourceLocation` (敌人生成预设RL)
- `special_rules: Dictionary` (锁门、钥匙卡需求、任务挂钩)

## 6) 实体：不同的技术实现 (按需求)

我们根据性能和开发速度明确区分实体类型的实现方式。

### 6.1 玩家 (混合型：节点驱动身体 + ECS状态)
**实现:**
- Godot节点：`CharacterBody2D` (移动、碰撞、动画)
- ECS实体：持有权威的“游戏状态”组件：
  - 生命值/耐力/流血/疼痛
  - 背包引用
  - 武器状态
  - 任务状态引用
- **玩家桥接器** 同步：
  - 节点位置 → ECS `C_Position`
  - ECS效果 → 节点动画/视觉特效/摄像机抖动

**为何混合?**
- 快速迭代手感和物理，同时利用可扩展的ECS逻辑处理战斗/状态/搜刮。

### 6.2 敌人 (ECS优先，节点轻量化)
**实现:**
- ECS实体处理AI、导航意图、战斗、生命值、状态。
- 可选的轻量级节点“视图”，用于精灵和受击闪光。
- 如果寻路使用Godot导航，则采用**服务**方式：
  - ECS请求实体路径 (批量)
  - 节点服务返回路径点
  - ECS跟随路径移动 (初期不每个敌人生成NavigationAgent2D)

**为何ECS优先?**
- 许多敌人同时活跃；区块激活和AI细节层次(LOD)变得直接。

### 6.3 抛射物 (仅ECS + 对象池)
**实现:**
- 每个抛射物为一个ECS实体：
  - 位置、速度、生命周期、伤害、穿透
- 碰撞处理：
  - 对物理空间进行轻量级射线查询 (批量) 或
  - 针对墙壁使用简化的网格碰撞，偶尔针对角色进行物理查询

**每个子弹不创建节点**，除非需要曳光弹视觉效果；使用池化视觉特效。

### 6.4 战利品容器 (主要为ECS + 节点标记)
**实现选项:**
- 在兴趣点场景中放置静态容器节点 (Area2D + 精灵)，并附加一个RL (`container_id`)
- 交互时，创建/激活一个ECS“容器状态实体”：
  - 已生成标记、生成的物品列表、锁定状态
- 这避免了预先创建成千上万个容器ECS实体。

### 6.5 门/锁 (节点物理 + ECS状态)
- 兴趣点片段中的门为节点，用于切换碰撞/动画
- ECS跟踪锁定状态 + 钥匙需求：
  - `required_key_tag` 或 `required_key_item_id` (RL)
- 交互触发节点动画和碰撞变化。

### 6.6 商人 (固定交互点；不是实体)
根据要求：**无商人实体**。
- 商人是安全屋中的**交互站**：
  - 节点：`TraderTerminal` / `ShopCounter`
  - 数据：`trader_id: ResourceLocation`
- 交易UI从注册表中定义的商人库存规则拉取：
  - 按标签划分的库存列表 + 刷新计时器
  - 引用物品ID/标签的以物易物规则

## 7) 安全屋“家园系统” (扩展需求)

安全屋是一个持久的元地图，包含：
- 建造与升级
- 生存改善
- 农业与工业
- 交易场所
- 地图扩展

### 7.1 家园模块 (注册表驱动)
每个家园模块通过RL和数据定义：
- `game:home/storage_upgrade_1`
- `game:home/med_station_1`
- `game:home/farm_plot_1`
- `game:home/workshop_1`
- `game:home/power_node_1`
- `game:home/trader_counter_1`
- `game:home/map_expansion_north_gate`

模块数据字段：
- 建造花费：物品标签需求 + 数量
- 前置条件：科技节点RL
- 提供：
  - 新制作配方
  - 新商人库存等级
  - 更大的仓库容量
  - 新的安全屋房间/区域
  - 生产能力变化

### 7.2 农业 (简单，可扩展)
- 农场地块随时间产生“原材料”。
- 加工站将原材料转化为有用的制作输入。
- 产出物供给制作和以物易物经济。

### 7.3 工业自动化 (轻量级网络，非物理传送带)
实现为图：
- 节点：生产者 / 处理器 / 存储 / 电源
- 边：运输链接
- 每个游戏刻：
  - 从上游缓冲区拉取输入
  - 执行配方
  - 将输出推送到下游
这使得自动化既有趣又在2D中可行，无需复杂的物理。

### 7.4 交易地点
交易在以下地点进行：
- `TraderTerminal` 站
- 通过扩展解锁的 `MarketStall` 站

## 8) Raid地图生成 (具体机制)

### 8.1 输入
- `raid_template_id: ResourceLocation` (基础TileMap集、生物群系标签、尺寸、难度)
- 随机种子
- 按生物群系/难度标签筛选的兴趣点池
- 用于敌人和容器的生成预设池

### 8.2 生成步骤 (推荐)
1. **生成基础地形**
   - 使用噪声 + 模板遮罩放置道路/河流/不可通行区域。
2. **定义候选区域**
   - 将地图划分为不同区域 (城市/工业区/田野/森林)。
3. **放置主要兴趣点**
   - 选择3–7个主要兴趣点 (地铁站、实验室、仓库、农场、检查站)。
   - 确保主要兴趣点之间保持最小距离。
   - 确保至少有一个“高风险高回报”的兴趣点。
4. **放置次要兴趣点**
   - 根据区域密度散布10–30个小兴趣点 (棚屋、车辆、小仓库)。
5. **连接通行路径**
   - 确保以下地点之间的可行走连通性：
     - 玩家出生区域
     - 至少2个撤离点
     - 主要兴趣点
6. **分配战利品 + 敌人配置**
   - 每个兴趣点获得：
     - `loot_profile_id` (战利品表RL)
     - `spawn_profile_id` (遭遇战RL)
7. **放置玩家出生点**
   - 从地图边缘或低风险区域的出生区域中选择。
   - 尽量远离价值最高的兴趣点，以促使玩家做出行进决策。
8. **放置撤离点 (关键)**
   - 生成3–6个撤离候选点：
     - **固定撤离点:** 1–2个始终可用，位置远离出生点，最好靠近地图边缘。
     - **条件撤离点:** 需要特定物品标签、开关激活、支付或时间窗口。
   - 规则：
     - 至少有一个撤离点距离出生点约60–75%的行程 (不能太近)。
     - 至少有一个撤离点位于地图的另一侧，以强制玩家规划路线。
     - 条件撤离点可以更近，但需要满足条件。
9. **放置任务标记**
   - 如果当前任务需要特殊物品：
     - 将它们生成在特定兴趣点或兴趣点场景内的指定标记点。
10. **区块激活设置**
    - 预计算区块元数据：
      - 哪些兴趣点与该区块相交
      - 进入该区块时激活哪些生成/战利品配置

### 8.3 撤离点类型示例 (数据驱动)
撤离点定义为注册表条目：
- `game:extract/edge_gate` (始终开启，倒计时8秒)
- `game:extract/paid_boat` (需要货币标签 `game:tag/currency`，倒计时5秒)
- `game:extract/keycard_door` (需要钥匙物品标签)
- `game:extract/power_switch` (需要先激活某个兴趣点的电源开关)

## 9) 存档系统 (多存档位)

### 9.1 存档位
多个档案槽：`slot_01`、`slot_02`等。
每个槽存储：
- 安全屋建造状态
- 仓库背包 (网格)
- 科技树解锁
- 商人声望/进度
- 任务进度状态
- 每个存档位的设置覆盖 (可选)

所有对内容的保存引用均使用 **ResourceLocation 字符串**。

### 9.2 内容清单兼容性
存档元数据包括：
- `save_version` (存档版本)
- `content_manifest_hash` (基于已注册RL键 + 版本的哈希)
缺失RL的回退：
- 将缺失物品替换为 `game:item_unknown_salvage` 或退还价值。

## 10) 里程碑 (修订版)

### 里程碑 1 — 最小化 Raid 循环 (搜刮/撤离)
- 基于RL的物品/标签/兴趣点/战利品表注册表
- 开放区域 + 兴趣点放置 + 撤离点放置
- 容器开启 → 延迟战利品生成 → 网格背包移动
- 1种武器类型 (通过标签) + 1种敌人类型
- 撤离/死亡 → 仓库更新
- 多存档位 创建/加载/删除

### 里程碑 2 — 安全屋家园系统
- 建造/升级核心模块 (存储、制作、农场地块、商人终端)
- 科技树消耗情报物品
- 基础制作 + 基础农业 + 首个自动化节点

### 里程碑 3 — 扩展与优化
- 区块激活和AI细节层次(LOD)分级
- 更多兴趣点和战利品表
- 更多敌人类型 (人类 + 非人类)
- 更多撤离点类型和任务挂钩
